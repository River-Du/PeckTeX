## 项目简介

**啄玛 (PeckTeX)** 是基于 **Python** 和 **PySide6** 框架构建的轻量级桌面应用。本应用通过调用多模态视觉大模型 (VLM) API，将图像中包含的数学公式、化学方程式、手写推导过程以及表格图表等内容，识别并转换为 **LaTeX** 等标记语言代码。

核心处理流程如下：

```text
截图/粘贴/文件导入 -> VLM API 识别 -> 结果流式输出 -> HTML/KaTeX 预览
```

应用 GUI 采用左右分栏结构：左侧为配置管理与动作区，右侧为识别结果、信息反馈与 AI 对话区。

## 功能特性

- **便捷图像输入**：支持桌面区域截图、系统剪贴板读取、本地文件选取/拖拽，以及用户图片文件夹的图片加载等。
- **自定义识别功能**：支持针对核心请求自定义各类提示词。预置功能包括 LaTeX 数学公式、MathML 数学公式、手写识别、化学方程式及通用识别等模板。
- **流式输出与本地预览**：服务端返回结果流式输出至编辑区，支持内容修改；输出公式代码可进行渲染预览，应用会生成本地临时 HTML 文件并调用 KaTeX 引擎进行呈现。
- **AI 对话修正**：内置 AI 多轮对话能力，可对当前的识别结果发起追问与订正，并将修正后的文本追加至结果区。
- **自动化能力**：支持导入图片后自动识别、识别后自动复制、批量识别、文本识别等功能。
- **历史记录管理**：所有识别结果展示在历史记录中，支持导入、导出记录文件，单条记录支持加载、复制、删除操作。
- **API 配置管理**：支持自定义 API 与多 API 管理；支持配置文件导入、重置与保存。

## 部署与启动

### 新手快速使用

如果你不打算改代码，建议直接下载 Release 版本使用：

1. 打开项目 Releases 页面：<https://github.com/River-Du/PeckTeX/releases>
2. 在最新版本中下载 Windows 压缩包 (通常为 `PeckTeX.zip`)。
3. 解压到一个固定目录 (例如 `D:\PeckTeX\`，不要放在临时下载目录)。
4. 进入解压后的目录，双击 `PeckTeX.exe` 启动。
5. 创建桌面快捷方式 (Windows)：右击 `PeckTeX.exe` -> `发送到` -> `桌面快捷方式`。

注意：

- 首次运行若出现 SmartScreen 提示，可点击“更多信息”后选择“仍要运行”。
- 软件目录下的`_internal` 文件夹是软件核心目录，请勿移动、修改或删除。
- 配置文件、用户图片文件与历史记录文件存放于 `userdata` 文件夹，建议不要删除。


### 环境依赖

**基础运行环境**
- 操作系统：Windows 10/11
- Python：>= 3.10

**核心依赖库**
- PySide6 >= 6.5.3
- openai >= 1.30.1
- httpx >= 0.27.0

### 源码启动

```bash
git clone https://github.com/River-Du/PeckTeX.git
cd PeckTeX
pip install -r requirements.txt
python main.py
```

仓库根目录提供 `run.bat`，可用于开发环境一键启动：

```bash
run.bat
```

### Windows 打包

仓库根目录提供 `build.bat`，可用于一键打包 (PyInstaller onedir 模式)：

```bat
build.bat --release
```

- 默认调试构建 (带控制台)，可通过参数切换发布构建 (`--debug` / `--release`)。
- 自动检查 Python、PyInstaller 与核心依赖是否可用。
- 自动清理旧的 `build/`、`dist/PeckTeX/` 与 `PeckTeX.spec` 产物后再构建。
- 打包后可执行文件路径：`dist/PeckTeX/PeckTeX.exe`。

## 软件使用介绍

1. **配置 API 和功能**：
   - 初次使用时，在界面左侧的 `设置` 区域，在 `平台` 中填写或选择一个 API 平台 (名称可自定义)，填写或确认平台的 `URL` 和对应的 API 密钥 `Key`，填写或选择一个平台支持的 `模型` (**必须是视觉 VLM 模型**)。
   - API 配置完成后，点击区域右上角的 `API 测试` 进行服务测试。如果右下角 `运行日志` 提示 “服务测试成功”，则说明 API 配置正确。
   - 在 `功能` 选框中填写或选择一个识别功能 (如“公式识别 LaTeX”)，并填写或确认下方 `提示词` 内容。
   - 点击 `保存配置` 按钮将当前设置写入 `userdata/config/config.json`，后续再次启动时将自动加载这些配置，可通过设置区域的选框选择相应的配置。

2. **输入图片**：
   - 点击左侧 `设置` 区域的 `截图` (全局快捷键 `Alt+S`) 发起屏幕截图，鼠标选取目标区域以输入图片。也可以通过系统剪贴板 `粘贴` (全局快捷键 `Ctrl+V`)、`文件` 导入、拖入图像至 `识别目标` 区域、用户 `图片文件夹` 加载，或鼠标右击 `识别目标` 区域等方式输入图片。
   - 图片输入后会在 `识别目标` 区域中显示预览图。点击图片可以查看大图。

3. **开始识别**：
   - 确认待识别的图片无误后，点击左下角的 `开始识别` (全局快捷键 `Alt+Return`) 发起识别任务。应用会将图片和提示词通过 API 调用发给视觉模型进行处理，生成的结果会流式输出到 `识别结果` 区域。
   - 在点击 `开始识别` 按钮前，可勾选旁边的 `自动识别`、`自动复制`、`连续识别`、`文本识别` 等复选框来调整识别行为。例如勾选 `自动识别` 和 `自动复制` 后，加载图片后会自动发起识别请求，并在识别成功后自动将结果复制到系统剪贴板。

4. **预览识别结果**：
   - 在成功输出 LaTeX 文本到 `识别结果` 区域文本框中后，点击 `预览` 按钮，可以在弹出的系统浏览器页面中确认 LaTeX 排版显示效果。
   - 若需要修改输出内容或进行测试，可直接点击 `识别结果` 区域的文本框进行编辑。
   - 确认无误后，可点击 `复制` 按钮将结果复制到系统剪贴板。

5. **查看历史记录**：
   - 在 `历史记录` 区域，可用鼠标滚轮浏览所有识别记录。所有识别成功的结果会按照时间顺序自动记录，也可在 `识别结果` 区域点击 `记录` 按钮手动记录当前区域中的内容。
   - 每一条记录可以单独进行 `复制`、`加载` (加载到识别结果区)、`删除`。也可以通过区域右上角的 `导入` 和 `导出` 按钮来管理历史记录文件。

6. **查看运行日志、与 AI 对话**：
   - 在 `运行日志` 区域，有一条 `状态栏` 可以查看动作响应结果和信息提示。点击 `状态栏` 可以将区域展开，查看更详细的运行日志，包括 AI 对话内容、详细信息提示等。
   - 展开区域后，可以在底部的 `对话输入框` 内直接与 AI 进行文本对话。输入内容后点击 `发送` 按钮，AI 会基于当前的对话上下文生成回复。对话回复选中文字，右击可以选择 `追加至识别结果` 来将回复内容追加到 `识别结果` 中。

7. **其他使用技巧**：
   - 每一个可交互按钮上都提供了悬浮提示，鼠标悬停在其上方即可查看功能说明。
   - 所有的配置内容 (API、功能、快捷键、复选框功能等) 都可以在 `userdata/config/config.json` 文件中直接编辑，修改后保存并重启应用即可生效。常用设置也可在界面的设置输入框中直接修改，点击 `保存配置`，同样直接作用于配置文件。

## 配置文件解析

项目运行所需的基础数据与业务预设均存储在 `userdata/config/config.json` 文件内。每次启动时，系统会校验数据，若检测到格式损坏会根据默认模板尝试恢复。

说明：

- 源码运行时，默认路径为：`项目根目录/userdata/config/config.json`。
- 打包运行时，默认路径为：`exe 同级目录/userdata/config/config.json`。

主要字段说明：

- `auto_recognize`：对应 `自动识别` 复选框。若为 `true`，加载新图像(截图、粘贴等)时自动触发识别请求。
- `auto_copy`：对应 `自动复制` 复选框。若为 `true`，识别成功后自动写入系统剪贴板。
- `continuous_recognition`：对应 `连续识别` 复选框。若为 `true`，点击开始识别后会自动识别系统预置图片文件夹中的图片，直到全部图片识别完成。
- `text_recognition`：对应 `文本识别` 复选框。若为 `true`，点击开始识别后不发送图片，改为发送识别结果区文本内容和提示词给 AI，适用于需要识别文本而非图片的场景。
- `continuous_chat`：若为 `true`，每次点击开始识别将不会清空对话上下文。适用于需要多轮对话来修正结果的场景。
- `language`：预留字段。
- `theme`：预留字段。
- `image_sort`：设置用户图片文件夹中文件的读取排序方式，支持 `"time"` (修改时间) 或 `"name"` (文件名)。
- `api_timeout`：网络请求的超时时限，单位为秒 (范围 `0.1 - 300`)。
- `max_history`：历史记录保留的记录上限。达到该阈值时自动覆盖最旧记录 (范围 `10-1000`)。
- `max_log`：运行日志保留的消息上限。达到该阈值时自动覆盖最旧记录 (范围 `10-1000`)。
- `shortcuts`：设置键盘快捷键绑定。包含截图 (`screenshot`)、粘贴 (`paste`) 及识别 (`recognize`) 字段。必须填入符合 Qt 规则的热键文本 (`Ctrl`, `Alt`, `Shift`, `Meta`, `Space`, `Esc`, `Return`, `Tab` ...)。
- `default`：启动时设置区的默认配置项，包含默认平台 (`platform`)、模型 (`model`) 和功能 (`function`)。启动时会自动加载到界面选项中。
- `platforms`：定义不同平台的 API 配置。可定义多个平台字段如 `openai`、`siliconflow` 等，每个平台字段包含 `api_url`、`api_key`、`models` 子字段，其中 `models` 列表应包含该平台支持的可用视觉 VLM 模型。
- `system_prompt`：设置大模型的系统提示词。
- `functions`：定义不同功能的提示词内容。可定义多个功能字段如 `formula_latex`、`handwriting` 等，每个功能字段应填写对应的提示词，指导模型理解用户需求和输出格式要求。

## 工程结构

### 架构分层

应用依赖 `PySide6` 组件栈实现，整体模块关系如下：

```text
theme.py -> gui_components.py -> gui.py <- settings.py / api_client.py / screenshot.py / renderer.py
main.py -> gui.py + theme.py
```

### 目录分层与职责

当前版本将“只读资源”和“可写用户数据”进行分离：

- 只读资源目录：`assets/` (图标等，打包时作为资源带入)。
- 可写用户数据目录：`userdata/` (配置、历史、临时文件、用户图片等)。

`userdata/` 用户文件内容：

- `userdata/config/config.json`：应用配置。
- `userdata/history/`：历史记录导入导出与自动存储。
- `userdata/images/`：用户图片文件夹。
- `userdata/temp/`：临时文件。

如果需要迁移使用环境，建议整体备份并迁移 `userdata/` 目录。

## 常见问题 (FAQ)

**Q：如果识别失败，该如何排查问题？** 

A：请确认 API 配置正确并且通过了 `API 测试`。未通过测试请展开 `运行日志` 查看详细错误提示信息，并尝试：1. 检查 API 配置；2. 检查 API Key 权限；3. 更换其他**视觉 VLM**模型；4. 检查网络连接；5. 稍后重试。如果 API 测试成功但识别失败，请检查错误提示信息。

**Q：如果输出结果中有符号识别错误，或漏掉下标该如何处理？**  

A：可直接在 `识别结果` 文本框中进行编辑。推荐使用追问对话功能：展开 `运行日志`，使用底部的聊天框向 AI 说明要求，如“请纠正上方代码，变量 k 应当修改为 i”，选中 AI 回复的正确结果文本，右击选择 `追加至识别结果`。

**Q：如何调用下拉列表中未预设的模型？**  

A：直接在 `模型` 下拉框中手写填入目标模型名称并回车确认即可。只要提供端的模型接口兼容 OpenAI 协议，程序均可正常下发请求。

**Q：是否支持批量识别？**

A：支持。可以点击 `图片文件夹` 按钮打开用户图片文件夹，将待识别的图片放入其中，然后，勾选 `连续识别` 选项后点击 `开始识别`，程序会自动依次识别文件夹中的图片，直到全部识别完成。

**Q：我没有 API Key，怎么办？**

A：可以访问相应的 API 服务商网站注册账号并获取 API Key，确保该 Key 具有访问视觉大模型 (VLM) 的权限。常见平台如 Siliconflow、OpenRouter、ModelScope 等模型聚合平台，以及 OpenAI、智谱 GLM 等原厂开发平台，都提供了视觉模型的 API 服务。请参阅对应平台官网的 API 说明。

**Q：是否支持切换语言？**

A：当前版本界面语言仅支持简体中文。
